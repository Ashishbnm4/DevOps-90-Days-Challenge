---

## What I Learned

- Managing system services

---

## Commands Practiced

### Process Commands
- `ps` → View running processes  
- `top` → Monitor system processes  

---

### Service Commands
- `systemctl status` → Check service status  
- `systemctl start` → Start a service  

---

### Log Commands
- `journalctl` → View system logs  

---

## Hands-on Practice

- Checked running processes using `ps` and `top`
- Monitored system services
- Viewed logs using `journalctl`
- Understood how Linux handles background services

## Key Takeaway

Understanding processes and services is essential for managing systems in DevOps and troubleshooting issues effectively.
